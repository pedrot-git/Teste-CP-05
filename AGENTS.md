# AGENTS.md

## Projeto
Landing page do aplicativo de música **Melodia**.

## Objetivo
Criar uma landing page moderna, premium, responsiva e visualmente impactante para apresentar o app Melodia.

O site deve transmitir:
- modernidade
- energia jovem
- tecnologia
- descoberta musical
- sofisticação
- imersão

## Público-alvo
- amantes de música
- jovens
- pessoas que buscam novas descobertas musicais

## Diferenciais do app
- qualidade de som superior
- playlists personalizadas
- descoberta de novos artistas
- interface intuitiva

## Stack obrigatória
- HTML5
- Tailwind CSS
- CSS customizado em arquivo separado
- JavaScript em arquivo separado
- GSAP para animações
- Font Awesome para ícones
- Google Fonts para tipografia

## Estrutura de arquivos
O projeto deve ser organizado em:
- `index.html`
- `style.css`
- `script.js`

Não unir CSS e JavaScript dentro do HTML, exceto quando for estritamente necessário.

## Direção visual
A landing page deve ter aparência de produto real e premium, não de template genérico.

### Estilo visual desejado
- dark mode premium
- gradientes vibrantes
- glow suave
- glassmorphism sutil
- cards arredondados
- sombras elegantes
- composição visual forte
- design clean
- boa hierarquia visual
- espaçamento refinado
- identidade marcante

### Sensação visual
Misturar:
- app de música premium
- startup moderna
- interface tecnológica
- estética jovem e sofisticada

## Paleta visual
Preferir:
- fundo escuro
- destaques em roxo, azul, rosa e ciano
- contraste forte
- botões chamativos mas elegantes

Evitar:
- cores apagadas demais
- combinações confusas
- excesso de tons sem hierarquia

## Tipografia
Escolher fontes do Google Fonts com aparência moderna e premium.

Regras:
- títulos fortes e marcantes
- textos limpos e legíveis
- ótima leitura no desktop e mobile

## Estrutura obrigatória da landing page

### 1. Header / Navbar
- navbar fixa no topo
- transparente no topo da página
- mudar ao scroll para fundo mais sólido com blur
- logo “Melodia”
- links de navegação
- CTA em destaque
- menu mobile funcional

### 2. Hero Section
- título principal: **Melodia: Sua Música, Sua Forma**
- subtítulo curto e persuasivo
- CTA principal: **Ouvir Agora**
- mockup, player ou composição visual do app
- elementos decorativos abstratos ligados à música
- visual impactante logo na primeira dobra

### 3. Benefícios
Destacar:
- qualidade de som superior
- playlists personalizadas
- descoberta de novos artistas
- interface intuitiva

Usar:
- ícones do Font Awesome
- cards modernos
- hover refinado

### 4. Funcionalidades
- apresentar funcionalidades em blocos visuais
- usar cards elegantes
- incluir elementos que lembrem produto real
- textos curtos, claros e modernos

### 5. Showcase visual do app
- criar uma seção visual forte mostrando o app em ação
- pode simular playlists, player, recomendação de artistas ou interface interna
- deve parecer uma vitrine premium do produto

### 6. Depoimentos
- cards com foto, nome e citação
- visual elegante e equilibrado

### 7. Formulário de contato / captação
- captar e-mails
- layout centralizado e responsivo
- aparência premium
- botão de envio chamativo
- feedback visual com JavaScript

### 8. Rodapé
- contato
- links úteis
- redes sociais
- política de privacidade
- visual coerente com o restante da página

## Animações
Usar GSAP com sofisticação e moderação.

Aplicar preferencialmente:
- entrada suave no hero
- stagger em textos, botões e cards
- reveal on scroll
- leve parallax em elementos decorativos
- microinterações elegantes
- transição da navbar ao scroll

Evitar:
- exagero
- animações distrativas
- efeitos pesados ou infantis

## Regras de código
- manter HTML semântico
- separar responsabilidades entre HTML, CSS e JS
- usar Tailwind para layout e responsividade
- usar CSS customizado para identidade visual e refinamento
- usar JavaScript para interações e GSAP
- manter código limpo e organizado
- evitar duplicação desnecessária
- preservar boa legibilidade
- comentar apenas o essencial

## Responsividade
O site deve funcionar muito bem em:
- desktop
- tablet
- mobile

Prioridades:
- boa adaptação de espaçamento
- boa hierarquia visual
- navbar responsiva
- botões clicáveis
- textos legíveis
- seções bem empilhadas no mobile

## Acessibilidade
- usar `alt` em imagens importantes
- manter contraste adequado
- usar botões e links claros
- evitar texto pequeno demais
- manter estrutura semântica correta

## O que evitar
- aparência genérica
- visual com cara de template pronto
- design simples demais
- excesso de texto
- excesso de elementos competindo entre si
- seções sem personalidade
- cards quadrados e sem refinamento
- animações exageradas
- visual poluído
- layout com cara escolar

## Prioridades do agente
Sempre priorizar, nesta ordem:
1. impacto visual
2. identidade moderna
3. responsividade
4. organização do código
5. fluidez das animações
6. clareza da estrutura

## Modo de trabalho esperado
Quando fizer alterações:
- analisar a estrutura atual antes de editar
- manter consistência visual entre as seções
- preferir mudanças incrementais
- não quebrar a responsividade
- não remover partes importantes sem necessidade
- melhorar estética e experiência sem complicar demais o código

## Resultado esperado
A página final deve parecer:
- um projeto de portfólio forte
- uma landing page real de startup musical
- moderna, premium, envolvente e memorável
